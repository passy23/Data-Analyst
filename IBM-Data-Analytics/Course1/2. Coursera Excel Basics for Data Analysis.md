## 1. Spreadsheet Fundamentals

- **Microsoft Excel** = most commonly used and fully-featured spreadsheet app. Alternatives: Google Sheets (free, collaborative, connects to online forms — best when multiple users collaborate), LibreOffice Calc (free, open source, has charts + pivot tables), Zoho Sheet, Apple Numbers.
- **Excel for the web has fewer advanced features** than the desktop version.
- Structure hierarchy: **Workbook → Worksheets → Ranges → Cells** (think: filing cabinet → drawers → folders → documents).
- **Cell reference:** column letter + row number, e.g. `M20`. A **range** ("array") uses `:` — `D9:D19` (column slice), `D9:H9` (row slice), `D9:H19` (block). Ranges let you refer to multiple cells in one formula.
- Spreadsheet limitations to know: performance degrades on very large datasets (hundreds of thousands of rows → consider a database), and a spreadsheet **doesn't preserve the sequence of transformation steps** — a key reason teams move to scripts/Power Query for reproducibility.

## 2. Formulas, Functions & Cell References

- Every formula starts with **`=`**.
- Formula components: values, **references**, operators, functions.
- **AutoFill** fills sequential series/patterns; the **Fill Handle** copies formulas (references adjust if relative).
- **AutoSum** shortcut: **Alt + =** · AutoSum dropdown includes: Sum, Average, **Count Numbers**, Max, Min.

### Reference types (⭐ interview + exam favorite)
| Type | Example | Behavior when copied |
|---|---|---|
| **Relative** (default) | `=A3+A5` | Adjusts with position |
| **Absolute** | `=$A$3+$A$5` | Locked completely |
| **Mixed** | `=A$1+$A6` | One part locked ($ before col = lock column; $ before row = lock row) |

- **F4** cycles relative → absolute → mixed.
- Deleting $ from someone's formula = bad idea — it breaks absolute references and produces wrong values.

### Function categories (Formulas tab → Function Library)
- **Financial:** ACCRINT, INTRATE · **Logical:** IF, IFS, AND, OR, NOT · **Text:** CONCAT, FIND, SEARCH, LEFT, RIGHT, LEN, TRIM, UPPER, LOWER, PROPER
- **Date & Time:** NOW, NETWORKDAYS, WEEKDAY · **Lookup & Reference:** VLOOKUP, HLOOKUP, XLOOKUP, SORTBY, AREAS · **Math & Trig:** SUM, SUMIF, SUMPRODUCT, POWER · **Statistical:** AVERAGE, COUNTIF, MEDIAN, MAX, MIN

### Formula errors (all start with `#`)
`#N/A` · `#NAME?` · `#NULL!` · `#REF!` · `#VALUE!` · `#DIV/0!` · Multiple `#####` = **column not wide enough** (or negative date/time).

## 3. Data Quality & Privacy

### Five traits of data quality (memorize: **ACRRT**)
1. **Accuracy** — free from errors
2. **Completeness** — all required data present
3. **Reliability** — trustworthy, so decisions aren't based on wrong info
4. **Relevance** — is the data actually necessary?
5. **Timeliness** — available and accessible when needed

### Three fundamentals of data privacy
**Confidentiality · Collection and Use · Compliance** — always check regulations for PII.

### Importing text data
**Text Import Wizard** (for .txt/.csv): indicate headers → select the **delimiter** → set the data format per column. After import, fix all column widths at once: select all columns → double-click one divider.

## 4. Cleaning Data in Excel — My Working Checklist

> [!tip] Data cleaning routine
> 1. Run spell check (**select the data first**, then Spelling button)
> 2. `Ctrl+↓` to catch the first empty row
> 3. **Remove empty rows:** filter for Blanks → select them → delete (empty rows cause formula errors)
> 4. **Remove duplicates:** select all → Data → Remove Duplicates (check "select all columns" + "my data has headers")
> 5. **Find & Replace** for known issues (e.g., replace 2 spaces with 1)
> 6. Standardize text case and trim whitespace
> 7. Fix date formats: Number Format → More Number Formats → Date → set Locale + Type

### Text-cleaning functions
| Function | Does |
|---|---|
| `=UPPER(A1)` / `=LOWER(A1)` | All caps / all lowercase |
| `=PROPER(A1)` | Sentence/title case |
| `=TRIM(A1)` | Removes excess whitespace |
| `=LEFT(A4,SEARCH(" ",A4,1))` | Extract **first name** (up to the first space) |
| `=RIGHT(A4,LEN(A4)-SEARCH(" ",A4,1))` | Extract **last name** (after the first space) |

- These functions need a **helper column** — afterwards copy and **Paste Values** (so formulas aren't pasted, just results).
- **Flash Fill** — pattern-based fill; can split first/last names or combine two columns. Also needs a helper column. **Text to Columns** is the other splitting tool.
- **Format Painter** — apply a format to the rest of a column.
- Conditional Formatting → Highlight Cells Rules → Duplicate Values = alternative duplicate finder (Remove Duplicates button is easier).

## 5. Analyzing Data: Filter, Sort & Key Functions

**Before shaping data, ask:** How big is the dataset? What filtering finds the needed info? How should it sort? What calculations are needed?

### Format as a Table — why
Automatic calculations while filtering · headers never disappear · banded rows · auto-expands with new rows · adds **filter drop-downs at the top of columns**.

### Sort & Filter
- Sort orders: A→Z / Z→A, Smallest→Largest, Oldest→Newest
- **Custom filters** > AutoFilters: filter above/below a value
- Clear ALL filters at once: **Data tab → Sort & Filter group → Clear**
- Wildcards work like SQL (`cars starting with…`)

### Logical & conditional functions
```excel
=IF(AD2>300,"Large",IF(AD2>100,"Medium",IF(AD2>0,"Small")))   ← nested IF
=IFS(AD2>300,"Large",AD2>100,"Medium",AD2>0,"Small")          ← cleaner IFS
=COUNTIF(N2:N195,"VISA")                                       ← count matching one criterion
=SUMIF(AF2:AF195,"Large",AD2:AD195)                            ← sum AD where AF="Large"
```
Each nested function needs its **own set of parentheses**.

### Lookups
```excel
=VLOOKUP(L3, C2:I113, 7, FALSE)
```
- `L3` = lookup value · `C2:I113` = lookup range · `7` = column number *within the range* · `FALSE` = exact match. Lock the range with F4 (`$C$2:$I$113`) before copying down.

| Function | Direction | Notes |
|---|---|---|
| **VLOOKUP** | Vertical (first column, left→right only) | Breaks if columns move |
| **HLOOKUP** | Horizontal (first row, top→bottom) | |
| **XLOOKUP** | **Any direction** | Replaces both; `if_not_found` for graceful errors |

### Multi-criteria versions
| Single criterion | Multi-criteria | Example |
|---|---|---|
| COUNTIF | **COUNTIFS** | `=COUNTIFS(B2:B6,"West",C2:C6,">500")` → count West sales over $500 |
| SUMIF | **SUMIFS** | `=SUMIFS(C2:C6,B2:B6,"West",C2:C6,">500")` → sum them |

XLOOKUP syntax: `=XLOOKUP(lookup_value, lookup_array, return_array, [if_not_found], [match_mode], [search_mode])` — e.g. `=XLOOKUP("Cherry",B2:B5,C2:C5)` → 2.00.

## 6. Pivot Tables ⭐ (my flagged skill gap — drill this)

**Why pivot tables:** manipulate data **without formulas** · quickly summarize large datasets · spot trends/patterns · dynamic (updates as source data changes) · feed charts and dashboards.

### Pre-pivot checklist
- [ ] Format data as a **table**
- [ ] Column headings correct, **only one header row** (headings become field names)
- [ ] Remove blank rows, columns, and cells
- [ ] Value fields formatted as **numbers** (not text)
- [ ] Date fields formatted as **dates** (not text)

### Working with pivots
- **PivotTable Fields pane** appears to the right when the pivot is selected (if hidden: PivotTable Tools → Options → Show group → Field List). Areas: Filters, Columns, Rows, Values.
- Add filters: drag a field to the **Filters area**
- Change Sum → Count: **Summarize Values By → Count**
- **Recommended Pivot Tables** = suggested field combinations based on selected data
- **Slicers** = on-screen graphical filter buttons (select a cell in the pivot first, then insert). Remove: right-click → Cut.
- **Timelines** = date-specific filter tool — much faster than manual date filters
- **Classic layout** allows dragging fields directly onto the grid
- Highlight a pivot → insert chart = instant pivot chart

## 8. Real-Life Data Project Checklist 📋

*Generalized from the Montgomery assignment — reusable for any Excel dataset at work, in the JDA program, or for portfolio projects.*

### Phase 0 — Before touching anything
- [ ] **Save an untouched backup copy** of the raw file (`filename_RAW.xlsx`) — never clean the original
- [ ] Confirm the required **output format and exact filename** (graders, clients, and pipelines are all picky)
- [ ] Check for **PII** — does this data fall under privacy rules? (confidentiality · collection & use · compliance)

### Phase 1 — Profile the data
- [ ] Convert to a working format (CSV → XLSX if needed)
- [ ] Fix column widths so everything is visible
- [ ] Count rows and columns — note them (you'll compare after cleaning)
- [ ] Skim for obvious structural problems: split columns, merged cells, multiple header rows

### Phase 2 — Clean
- [ ] Remove **empty rows** (filter → Blanks → delete)
- [ ] Remove **duplicates** (all columns checked, "my data has headers" on)
- [ ] Run **spell check** on selected data
- [ ] Kill **double spaces** (Find & Replace until 0 results) and `=TRIM()` stray whitespace
- [ ] Standardize **text case** (UPPER/LOWER/PROPER in helper column → Paste Values)
- [ ] Standardize **dates** (Format Cells → Date → correct locale) and confirm numbers aren't stored as text
- [ ] Fix split/combined columns (`=A1&" "&B1`, Flash Fill, or Text to Columns)
- [ ] Delete helper columns after Paste Values

### Phase 3 — Validate the cleaning
- [ ] Row count vs. Phase 1 — can I explain every removed row?
- [ ] Sanity aggregates: SUM / COUNT / MIN / MAX — does COUNT match expected rows? Is MIN/MAX plausible or an outlier to investigate?
- [ ] Spot-check 5–10 random rows against the raw backup

### Phase 4 — Analyze
- [ ] **Format as Table** before anything else
- [ ] Pivot pre-flight: one header row · no blanks · values as numbers · dates as dates
- [ ] Build pivots to answer the actual business question; sort so the story is obvious (usually descending)
- [ ] Name worksheets clearly and put them in a logical order

### Phase 5 — Deliver (the step that cost me 8 points)
- [ ] Save/download with the **exact required filename**
- [ ] **Reopen the downloaded file and click through every single sheet** — verify pivots, charts, and formulas actually exported
- [ ] Confirm no leftover helper columns, stray sheets, or filters hiding data
- [ ] Document what was cleaned and why (Excel keeps no history — my notes are the audit trail)

## 🎯 Interview Questions — JDA Path (Excel focus)

Excel questions are near-guaranteed in Data Analyst interviews. Practice these with real examples from the NPower labs and the fleet inventory final project.

**Q1. Explain relative, absolute, and mixed cell references. When would you use each?**
> Relative (`A1`) adjusts as the formula is copied — the default, right for row-by-row calculations. Absolute (`$A$1`) is fully locked — used when every formula must point at one fixed cell, like a tax rate or exchange rate. Mixed (`A$1` or `$A1`) locks only the row or column — essential for two-dimensional fill patterns like a multiplication grid. F4 cycles through them.

**Q2. VLOOKUP vs. XLOOKUP — which do you prefer and why?**
> XLOOKUP where available: it searches any direction, isn't broken by inserting columns (it uses separate lookup and return arrays instead of a column index), and has built-in `if_not_found` error handling. VLOOKUP is still worth knowing because legacy workbooks are full of it — I'd explain that it searches the *first* column only, left to right, and needs FALSE for exact match.

**Q3. Your VLOOKUP returns #N/A. How do you troubleshoot?**
> Check: (1) does the value actually exist in the first column of the range; (2) data type mismatch — numbers stored as text is the classic culprit, fixed with VALUE() or re-formatting; (3) trailing spaces — wrap in TRIM; (4) is the range locked with absolute references before I copied it down; (5) exact vs. approximate match flag.

**Q4. Walk me through how you'd clean a messy dataset in Excel.**
> My routine: spell check the selected data, jump with Ctrl+↓ to find gaps, filter for Blanks and delete empty rows, Remove Duplicates with all columns checked, Find & Replace for known inconsistencies like double spaces, TRIM whitespace, standardize text case with UPPER/LOWER/PROPER in a helper column and Paste Values back, and normalize date formats via Format Cells with the right locale. I document each step — Excel doesn't record transformation history, so my notes are the audit trail.

**Q5. What's the difference between COUNTIF and COUNTIFS? Give an example.**
> COUNTIF handles one criterion — `=COUNTIF(N2:N195,"VISA")` counts Visa transactions. COUNTIFS handles multiple — `=COUNTIFS(Region,"West",Amount,">500")` counts West-region sales over $500. Same relationship between SUMIF and SUMIFS.

**Q6. What is a pivot table and when would you use one?**
> A tool that summarizes large datasets — grouping, counting, summing, averaging — without writing formulas, by dragging fields into rows, columns, values, and filters. I'd use one whenever a stakeholder asks an aggregation question: sales by region by quarter, headcount by department. It's dynamic, so it updates as source data changes, and it feeds charts directly.

**Q7. How do you prepare data before building a pivot table?**
> Format the range as a table, ensure exactly one header row with correct headings (they become field names), remove blank rows/columns/cells, and confirm values are stored as numbers and dates as dates — text-formatted numbers silently break aggregation.

**Q8. What are slicers and timelines?**
> Both are visual filters for pivot tables. Slicers are clickable buttons for filtering any field — great for dashboards non-technical users touch. Timelines are the same idea specialized for dates, letting you drag across months or quarters instead of building date filters manually.

**Q9. How would you split a full-name column into first and last name?**
> Three ways, fastest first: Flash Fill (type the pattern once, Excel infers it), Text to Columns with space as the delimiter, or formulas for a dynamic solution — `=LEFT(A4,SEARCH(" ",A4,1))` for first name and `=RIGHT(A4,LEN(A4)-SEARCH(" ",A4,1))` for last. Formulas go in helper columns, then Paste Values.

**Q10. A cell shows #####. What's wrong?**
> Usually nothing is wrong with the data — the column is just too narrow to display it. Widen it or double-click the divider. The other cause is a negative date/time value.

**Q11. What are the five traits of data quality?**
> Accuracy, Completeness, Reliability, Relevance, and Timeliness. In practice I'd add: relevance means asking whether the data is really necessary at all, and timeliness means it's available and accessible when the decision needs it.

**Q12. When is Excel the wrong tool?**
> Very large datasets — hundreds of thousands of rows strain performance and stability, so a database plus SQL, or Python/pandas, is the better fit. Also when reproducibility matters: Excel doesn't preserve the sequence of cleaning steps, whereas a script does. Part of an analyst's judgment is knowing when to graduate from the spreadsheet.

**Q13. Explain nested IF vs. IFS.**
> Nested IF chains conditions inside each other — `=IF(x>300,"Large",IF(x>100,"Medium","Small"))` — each IF needing its own parentheses, which gets error-prone deep down. IFS flattens it: `=IFS(x>300,"Large",x>100,"Medium",x>0,"Small")` — evaluated left to right, first true condition wins. Cleaner and easier to audit.

**Q14. Why use Paste Values?**
> When helper-column formulas have done their job, pasting values replaces the formulas with their results — so the clean data no longer depends on the helper column (which I can then delete) and won't recalculate or break if references change.

**Q15. Tell me about a data cleaning project you've done.** *(⭐ STAR story — the Montgomery fleet project)*
> **Situation:** As part of my IBM Data Analyst certification I played a junior analyst for a local government office, handed a raw fleet-inventory CSV. **Task:** clean it and produce a pivot-table analysis of equipment counts by department. **Action:** I converted to XLSX, removed empty rows via blank filtering, deduplicated, fixed spelling and double spaces, and repaired department names that had been split across two columns — I used `=A1&" "&B1` with Paste Values instead of Flash Fill because the formula is deterministic on irregular data. I validated with AutoSum sanity checks (49 records, sum 1,582, max 379), then built three pivot views: totals by department, department→vehicle-type drill-down, and the inverted vehicle-type→department view. **Result:** the cleaning scored perfect marks; and I learned a delivery lesson the hard way — my exported file lost the pivot sheets, so now I never submit a file without reopening the downloaded copy first.

**Q16. When would you use HLOOKUP instead of VLOOKUP?**
> When the comparison values run **across a row at the top** of the table instead of down the left column. V = vertical, searching down the first column; H = horizontal, searching across the first row. In new work I'd use XLOOKUP for either orientation, but I need to read both in legacy workbooks. *(← the one quiz question I missed — drilled now.)*

**Q17. How do you verify your cleaning actually worked before delivering?**
> Three layers. First, counts: compare row counts before and after — I should be able to explain every removed row. Second, sanity aggregates: SUM, COUNT, MIN, MAX — if COUNT doesn't match the expected records there are hidden blanks or text-formatted numbers; an implausible MIN/MAX flags an outlier or entry error. Third, spot-check random rows against the untouched raw backup I always keep. And the final gate: reopen the actual deliverable file and inspect every sheet — what's in my editing session isn't necessarily what exported.

**Q18. What are the different ways to combine text from two columns?**
> Four: the `&` operator — `=A1&" "&B1` — my default because it's explicit and deterministic; the CONCAT function, same result in function form; Flash Fill (Ctrl+E), which infers the pattern from an example — fast, but it can guess wrong on irregular data; and TEXTJOIN when combining many columns with a delimiter. Whichever I use, I finish with Paste Values and delete the sources.

**Q19. What are the risks of working in Excel for the web vs. desktop?**
> Fewer advanced features, and a subtle delivery trap: web auto-saves the cloud copy, which gives false confidence — the *downloaded export* is a separate artifact that can silently drop content (I lost pivot tables to exactly this). Desktop is more feature-complete and what-you-save-is-what-you-send. On web, always reopen the download before sharing.

**Q20. You receive a dataset from another team. What's the first thing you do?**
> Before any edits: save an untouched backup of the raw file, then profile it — row/column counts, structural issues like merged cells or split columns, data types, and whether it contains PII that triggers privacy obligations. Cleaning starts only after I know what "correct" should look like, and every change gets documented since Excel keeps no transformation history.
