# 📊 What is Data Analytics?

Data Analytics is the process of examining raw data to discover meaningful patterns, trends, and insights that support informed decision-making.

Organizations rely on data analytics to:

* Improve operational efficiency
* Understand customer behavior
* Reduce costs
* Optimize business processes
* Predict future outcomes
* Support strategic planning
* Measure organizational performance

Today, data has become one of the most valuable business assets, enabling organizations to make evidence-based decisions rather than relying solely on intuition.

---

# 📈 Types of Analytics

Data analytics can be categorized into four primary types:

| Type                       | Purpose                         | Example                                |
| -------------------------- | ------------------------------- | -------------------------------------- |
| **Descriptive Analytics**  | Explains what happened          | Monthly sales reports                  |
| **Diagnostic Analytics**   | Explains why something happened | Root cause analysis of declining sales |
| **Predictive Analytics**   | Forecasts future events         | Predicting customer churn              |
| **Prescriptive Analytics** | Recommends actions              | Product recommendation systems         |

These categories represent increasing levels of analytical maturity, progressing from understanding historical data to making intelligent recommendations.

# 👨‍💼 Role of a Data Analyst

A Data Analyst serves as the bridge between raw data and business decisions.

Typical responsibilities include:

* Collecting data from multiple sources
* Cleaning and preparing datasets
* Writing SQL queries
* Performing exploratory data analysis (EDA)
* Creating visualizations and dashboards
* Identifying trends and anomalies
* Communicating insights to stakeholders
* Supporting data-driven decision-making

The role requires both technical expertise and strong communication skills.

# 🔄 The Data Analytics Lifecycle

A typical analytics workflow follows these stages:

1. Business Problem Definition
2. Data Collection
3. Data Cleaning
4. Data Wrangling
5. Data Storage
6. Data Exploration
7. Data Analysis
8. Data Visualization
9. Communication of Insights
10. Business Decision & Continuous Improvement

This lifecycle emphasizes that analytics is an iterative process, where findings often lead to new questions and additional analysis.

# 🗂 Types of Data

Understanding data structures is essential because different data types require different storage and processing techniques.

## Structured Data

Structured data follows a predefined schema and is typically organized into rows and columns.

Examples include:

* Relational databases
* Excel spreadsheets
* Transaction records

Characteristics:

* Highly organized
* Easily searchable
* SQL compatible
* Ideal for reporting

---

## Semi-Structured Data

Semi-structured data contains organizational markers but does not conform to a rigid relational schema.

Examples include:

* JSON
* XML
* Emails
* Server logs

Characteristics:

* Flexible structure
* Metadata-driven
* Frequently exchanged through APIs

---

## Unstructured Data

Unstructured data lacks a predefined format.

Examples include:

* Images
* Videos
* Audio recordings
* PDFs
* Social media posts

Characteristics:

* Largest proportion of enterprise data
* More difficult to analyze
* Often requires AI and machine learning techniques

---

# 🗄 Data Storage Technologies

Organizations store information using several repository types, each designed for different purposes.

## Relational Databases (RDBMS)

Relational databases organize information into tables connected by relationships.

Common systems include:

* IBM Db2
* MySQL
* PostgreSQL
* Microsoft SQL Server
* Oracle Database

Advantages:

* ACID compliance
* High consistency
* SQL support
* Strong data integrity

---

## NoSQL Databases

Designed for scalability and flexibility when handling large or rapidly changing datasets.

Major categories include:

* Key-Value Stores
* Document Databases
* Column-Family Databases
* Graph Databases

NoSQL databases are commonly used in web applications, IoT, social networks, and real-time systems.

---

## Data Warehouse

A centralized repository that stores cleaned, integrated, and structured data from multiple operational systems.

Primary purpose:

* Business Intelligence
* Reporting
* Historical analysis
* Executive dashboards

---

## Data Mart

A specialized subset of a data warehouse focused on a specific department or business function, such as finance, marketing, or human resources.

Benefits:

* Faster access
* Department-specific analytics
* Simplified reporting

---

## Data Lake

A Data Lake stores raw data in its original format, including:

* Structured data
* Semi-structured data
* Unstructured data

It is particularly valuable for big data analytics and machine learning workloads.

# 📊 Big Data

Traditional database technologies struggle with today's massive volumes of information.

Big Data is commonly described using the **Five Vs**:

| Characteristic | Description                             |
| -------------- | --------------------------------------- |
| Volume         | Massive amounts of data                 |
| Velocity       | Speed of data generation and processing |
| Variety        | Multiple data formats                   |
| Veracity       | Data quality and reliability            |
| Value          | Business usefulness of data             |

# ⚙ Big Data Technologies
